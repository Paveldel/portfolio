Het doel van deze e-portfolio is om een beeld te geven van wie ik ben, wat mijn interesses zijn en welke ervaringen ik al heb opgedaan. Een e-portfolio kan echter op veel verschillende manieren worden ontworpen. In deze post wil ik kort toelichten hoe ik deze website heb opgebouwd en waarom ik voor deze aanpak heb gekozen.

Mijn eerste idee was om een WordPress-site te gebruiken. Tijdens het Adria-project had ik al ervaring opgedaan met deze technologie. WordPress is zeer geschikt voor het maken van blogs en portfolio's, maar er was toch iets dat me tegenhield. Het platform neemt namelijk veel technische aspecten uit handen en vereist slechts een beperkte hoeveelheid programmeerwerk. Hoewel dat een voordeel kan zijn, vond ik het jammer om de kans te laten liggen om iets nieuws te leren voor deze portfolio. Daarom besloot ik op zoek te gaan naar een alternatieve oplossing.

Ruby was een programmeertaal waar ik tot dan toe nog geen ervaring mee had. De taal is vooral bekend door het succes van Shopify, maar wordt ook gebruikt voor tal van andere toepassingen. Een van de bekendste Ruby-libraries is Jekyll, en dat is precies de technologie waarop deze website gebouwd is.

Jekyll is een relatief eenvoudig framework waarmee statische websites kunnen worden gemaakt. Vervolgens kan je de website verder personaliseren door gebruik te maken van thema's of door zelf extra code te schrijven voor de vormgeving. Ik heb verschillende thema's bekeken, maar uiteindelijk gekozen voor Chirpy. Hoewel Chirpy in de eerste plaats gericht is op blogs, biedt het voldoende flexibiliteit om ook eigen pagina's te ontwerpen, zoals de 'About'-pagina die ik voor deze portfolio nodig had.

Daarnaast bevat Chirpy verschillende handige functies, zoals een zoekbalk, ondersteuning voor categorieën, een light- en dark mode en de mogelijkheid om berichten te delen via sociale media. Ook de standaardvormgeving sprak me aan. Het ontwerp oogt professioneel, is minimalistisch en blijft tegelijk overzichtelijk en gebruiksvriendelijk op verschillende schermformaten.

Ook voor het publiceren van een website bestaan er verschillende mogelijkheden. Voor deze portfolio heb ik ervoor gekozen om GitHub Pages uit te proberen. Deze dienst van GitHub maakt het mogelijk om websites te hosten op een GitHub-URL en ondersteunt verschillende frameworks, waaronder Jekyll. Een groot voordeel is dat nieuwe versies van de website automatisch worden uitgerold wanneer er code naar een bepaalde branch wordt gepusht. Dat is bijzonder handig voor een blog, omdat nieuwe berichten hierdoor automatisch online verschijnen zonder extra stappen.
